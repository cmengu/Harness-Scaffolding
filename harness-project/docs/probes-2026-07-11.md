# Phase-1 probe report — 11 Jul 2026

Versions probed: claude 2.1.207 · codex-cli 0.142.5. Artifacts in session scratchpad
(`probes/p1–p9`). **Verdict: per-head sessions are GO — plan B (tools round-0-only)
not needed.**

## claude head

| Question | Answer |
|---|---|
| Streaming deltas headless? | ✅ `-p --output-format stream-json --include-partial-messages` — **requires `--verbose`**. Events: `stream_event` wrapping API `content_block_start/delta/stop`, `message_delta`. |
| Thinking trigger under `-p`? | `MAX_THINKING_TOKENS` env + a "think hard/ultrathink" phrase → thinking block fires (`thinking_tokens` in `message_delta` usage). |
| Thinking TEXT visible? | ❌ **REDACTED headless.** `thinking_delta` events arrive with EMPTY text; transcript jsonl also stores empty thinking. Tape must render a `✳ thinking… (n tokens)` pulse, never text. |
| `total_cost_usd`? | ✅ on the final `result` event, plus `session_id`, `num_turns`, `duration_ms`. |
| Session mint + resume? | ✅ `--session-id <uuid>` then `-p --resume <uuid>`. Remembers tool-derived facts (read a file round 0, recalled its secret on resume with tools off). Resumed turn ~5× cheaper (cache): $0.128 → $0.024. Works combined with stream-json. |
| Prompt delivery | via **stdin** (variadic `--allowedTools` eats a positional prompt — pre-existing pin, still true). |

## codex head

| Question | Answer |
|---|---|
| `exec --json` schema | JSONL: `thread.started {thread_id}` → `turn.started` → `item.started/item.completed {item: {type, text}}` → `turn.completed {usage}`. Item types seen: `agent_message`, `reasoning`, `web_search`. |
| Session id | `thread_id` on `thread.started` (also = resume handle). |
| Resume | ✅ `codex exec resume <thread_id> "<prompt>"` — remembered the round-0 codeword; same thread_id re-emitted. |
| Reasoning summaries | ✅ visible as `item.completed type=reasoning` with real text when `-c model_reasoning_effort='"high"' -c model_reasoning_summary='"detailed"'`. |
| Web search | ✅ `-c tools.web_search=true` (the `--search` flag exists only on interactive `codex`, NOT `exec`). Emits `web_search` items with the query. |
| Cost | ❌ no dollar figure — `turn.completed.usage` is tokens only (`input/cached_input/output/reasoning_output`). Report codex cost in tokens. |
| Gotchas | Always pass explicit stdin (`</dev/null`) or it blocks on "Reading additional input from stdin…"; outside a trusted git dir needs `--skip-git-repo-check`. |

## Design consequences

- **Step 2 builds the resume path**: claude `--session-id`/`--resume`, codex `thread_id`/`exec resume`. Preamble replay remains only as the degraded fallback when a resume call errors.
- **Glass-box asymmetry confirmed** (accepted 10 Jul): codex contributes readable reasoning summaries; claude contributes only a thinking pulse with token count.
- Round-1 resumed calls are materially cheaper (cache) — the 2-calls-per-head default costs less than 2× round 0.
